# Proxmox Helper Toolkit `.pht` Format Documentation

## Overview

The `.pht` format is the configuration format used by the Proxmox Helper Toolkit.

A `.pht` file defines an interactive menu entry and tells the toolkit:

- What the entry is called
- What actions are available
- Which commands should be executed
- Which arguments should be passed
- What user input should be collected
- Which optional flags exist
- Whether confirmation is required

The `.pht` file does not contain the actual program logic. It only describes the user interface and command execution.

Each `.pht` file inside the `pht-files` directory becomes a selectable entry.

Example structure:

    pht-files/
    ├── auto-nas.pht
    ├── update-cluster.pht
    └── backup-manager.pht


---

# File Structure

A `.pht` file contains the following sections:

1. Metadata
2. Actions
3. Fields
4. Options
5. Flags
6. Confirmations


Basic example:

    title = "Example Tool"
    description = "Example description"

    [action]

    name = "Run Example"
    command = "example-command"

    [/action]


---

# Values

Values must be quoted.

    title = "My Tool"

For sub-quotation:

    command = "My-Tool '/mnt/path with space/Share'"


---

# Metadata

Metadata defines the menu entry.

Metadata must appear before any `[action]` blocks.


Example:

    title = "Auto NAS"
    description = "Automatically mount USB drives"


## Metadata Properties

| Property | Required | Description |
|---|---|---|
| title | No | Entry display name |
| description | No | Description shown below title |
| hidden | No | Hide entry from menus |


## Hidden Entries

Entries can be hidden:

    title = "Internal Tool"
    hidden = true

Hidden entries are not shown in the menu.


---

# Actions

Actions define commands that can be executed.

Each action appears in the action menu.


Syntax:

    [action]

    name = "Action Name"
    command = "command-name"
    arguments = "arguments"

    [/action]


## Action Properties

| Property | Required | Description |
|---|---|---|
| name | Yes | Display name |
| command | Yes | Executable command |
| arguments | No | Static command arguments |
| hidden | No | Hide action |
| default | No | Automatically select action |


---

# Basic Action Example

    [action]

    name = "Show Status"
    command = "systemctl"
    arguments = "status nginx"

    [/action]


Runs:

    systemctl status nginx


---

# Default Actions

Actions can automatically be selected.

Example:

    [action]

    name = "Update System"
    command = "apt"
    arguments = "update"
    default = true

    [/action]


---

# Hidden Actions

Actions can be hidden:

    [action]

    name = "Developer Debug"
    command = "debug-tool"
    hidden = true

    [/action]


---

# Arguments

Arguments are appended after the command.

Example:

    [action]

    name = "Restart Service"
    command = "systemctl"
    arguments = "restart nginx"

    [/action]


Runs:

    systemctl restart nginx


---

# Arguments With Spaces

Arguments containing spaces must be quoted.

Example:

    [action]

    name = "Send Message"
    command = "notify"

    arguments = "--message 'hello world'"

    [/action]


Execution:

    notify --message "hello world"


The argument parser keeps quoted values together.


---

# Fields

Fields collect user input before execution.

Syntax:

    [field]

    name = "Field Name"
    type = "input"

    [/field]


Fields are appended after action arguments.


Example:

    [action]

    name = "Mount USB Drive"
    command = "auto-nas"
    arguments = "mount"


    [field]

    name = "Device"
    type = "input"

    [/field]


    [/action]


Execution:

    auto-nas mount /dev/sdb1


---

# Field Properties

| Property | Required | Description |
|---|---|---|
| name | Yes | Input label |
| type | Yes | Input type |


---

# Field Types


## input

Normal text input.

Example:

    [field]

    name = "Username"
    type = "input"

    [/field]


---

## password

Hidden password input.

Example:

    [field]

    name = "Password"
    type = "password"

    [/field]


---

## path

Filesystem path input.

Example:

    [field]

    name = "Backup Location"
    type = "path"

    [/field]


---

## ipv4

IPv4 address input.

Example:

    [field]

    name = "Server IP"
    type = "ipv4"

    [/field]


---

## number

Numeric input.

Example:

    [field]

    name = "Port"
    type = "number"

    [/field]


---

## list

A selectable list.

Example:

    [field]

    name = "Log Type"
    type = "list"


    [option]

    name = "Live"
    value = "live"

    [/option]


    [option]

    name = "Recent"
    value = "recent"

    [/option]


    [/field]


The selected option value is passed to the command.


---

# Options

Options are only valid inside list fields.


Syntax:

    [option]

    name = "Display Name"
    value = "command-value"

    [/option]


## Option Properties

| Property | Required | Description |
|---|---|---|
| name | Yes | Menu display text |
| value | Yes | Command argument value |


---

# Flags

Flags are optional arguments controlled by confirmation prompts.

Syntax:

    [flag]

    name = "Enable Feature"
    arguments = "--enable"

    [/flag]


If enabled, the arguments are added before normal arguments.


Example:

    [action]

    name = "Update Cluster"
    command = "update-cluster"


    [flag]

    name = "Update nodes in parallel"
    arguments = "--parallel"

    [/flag]


    [/action]


Execution:

    update-cluster --parallel


---

# Confirmations

Confirmations prevent accidental actions.

Syntax:

    [confirm]

    message = "Confirmation message"

    [/confirm]


Example:

    [action]

    name = "Remove Auto NAS"
    command = "auto-nas"
    arguments = "remove"


    [confirm]

    message = "Remove Auto NAS?"

    [/confirm]


    [/action]


---

# Command Construction Order

Commands are built in this order:

1. Flags
2. Action arguments
3. Field values


Example:

    [action]

    name = "Copy File"

    command = "cp"

    arguments = "-v"


    [flag]

    name = "Force overwrite"

    arguments = "-f"

    [/flag]


    [field]

    name = "Source"

    type = "path"

    [/field]


    [field]

    name = "Destination"

    type = "path"

    [/field]


    [/action]


Generated command:

    cp -f -v source destination


---

# Recommended Naming

Use lowercase names with hyphens.

Recommended:

    auto-nas.pht
    update-cluster.pht
    backup-manager.pht


Avoid:

    My Script.pht
    test!!.pht
